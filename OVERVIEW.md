# 皮蛋一号 (Pidan1) 项目概览

## 项目简介

皮蛋一号是一个基于 FPGA 的 SoC 系统，集成了：
- **VexRiscv-SMP 处理器**：Linux capable RISC-V 多核处理器
- **BitNet 加速器**：1-bit 量化神经网络推理硬件加速
- **完整的外设支持**：DDR3、以太网、UART 等

## 技术架构

### 硬件平台
- FPGA: Xilinx Artix-7 200T (XC7A200T-FBG484)
- 内存: 512MB DDR3 (MT41K256M16)
- 网络: RTL8211E Gigabit Ethernet PHY
- 调试: CH340 USB-UART

### 软件栈
- 操作系统: Linux (RISC-V)
- 引导程序: OpenSBI
- 工具链: LiteX + Vivado 2024.2

## 核心创新：BitNet 加速器

### 设计特点

1. **1-bit 量化**
   - 权重量化为 {-1, 0, +1}
   - 2-bit 编码存储（00→0, 01→+1, 10→-1）
   - 大幅减少存储和计算开销

2. **高效存储**
   - 权重区：256 words (每个 word 存储 16 个权重)
   - 输入区：16 words (4 个 8-bit 元素打包)
   - 输出区：64 words (32-bit 有符号结果)
   - 总共 4KB 地址空间

3. **Wishbone 总线接口**
   - 标准 32-bit Wishbone 从设备
   - 映射到非缓存区域 0x80002000
   - 易于集成到 LiteX SoC

4. **简单的控制流程**
   ```
   配置维度 → 写入权重 → 写入输入 → 启动 → 等待完成 → 读取结果
   ```

### 性能指标（预期）

- 最大矩阵：64×64
- 时钟频率：100 MHz
- 单次计算延迟：~64 周期（对于 64×64 矩阵）
- 吞吐量：~1.56 GOPS (100M × 64 / 4096)

## 寄存器映射详解

### 控制和状态寄存器

| 偏移 | 名称 | 类型 | 位域 | 说明 |
|------|------|------|------|------|
| 0x000 | CTRL | W | [0] start | 写1启动计算 |
|  |  |  | [1] clear | 写1清除状态 |
| 0x004 | STATUS | R | [0] done | 计算完成标志 |
|  |  |  | [1] busy | 计算进行中标志 |
| 0x008 | SIZE_M | RW | [5:0] | 矩阵行数 (1~64) |
| 0x00C | SIZE_N | RW | [5:0] | 矩阵列数 (1~64) |

### 数据缓冲区

| 偏移 | 名称 | 类型 | 大小 | 说明 |
|------|------|------|------|------|
| 0x010~0x40F | WEIGHT | RW | 256 × 4B | 权重存储（2-bit编码） |
| 0x410~0x44F | INPUT | RW | 16 × 4B | 输入向量（8-bit打包） |
| 0x450~0x54F | OUTPUT | R | 64 × 4B | 输出结果（32-bit有符号） |

### 权重编码格式

每个 32-bit word 存储 16 个 2-bit 编码的权重：

```
Word[31:0] = {w15, w14, ..., w1, w0}
每个 wi 占用 2 bits:
  00 → 权重值 0
  01 → 权重值 +1
  10 → 权重值 -1
  11 → 保留（未使用）
```

### 输入打包格式

每个 32-bit word 存储 4 个 8-bit 输入：

```
Word[31:0] = {in3[7:0], in2[7:0], in1[7:0], in0[7:0]}
```

## 使用示例

### C 语言示例（伪代码）

```c
// 1. 初始化
volatile uint32_t *accel = mmap(..., 0x80002000, ...);

// 2. 配置维度
accel[0x008/4] = 32;  // M = 32
accel[0x00C/4] = 32;  // N = 32

// 3. 写入权重（已编码）
for (int i = 0; i < 256; i++) {
    accel[(0x010 + i*4)/4] = encoded_weights[i];
}

// 4. 写入输入
for (int i = 0; i < 16; i++) {
    accel[(0x410 + i*4)/4] = packed_input[i];
}

// 5. 启动计算
accel[0x000/4] = 0x1;  // start

// 6. 等待完成
while (!(accel[0x004/4] & 0x1));  // wait for done

// 7. 读取结果
for (int i = 0; i < 32; i++) {
    result[i] = accel[(0x450 + i*4)/4];
}
```

## 构建流程

### 1. 生成 Verilog RTL

LiteX 将 Python 描述转换为 Verilog：
- `pidan1_soc.py` → Migen → Verilog
- 集成 `bitnet_accel.v`
- 生成约束文件 (XDC)

### 2. Vivado 综合与实现

```
Verilog → 综合 → 网表 → 布局布线 → Bitstream
```

### 3. FPGA 配置

- SRAM 模式：临时加载（JTAG）
- Flash 模式：永久烧录（SPI Flash）

### 4. Linux 引导

```
OpenSBI (M-mode) → Kernel (S-mode) → User Space
```

## 性能分析

### 资源使用（估算）

| 资源 | 使用量 | 百分比 |
|------|--------|--------|
| LUT | ~80K | ~37% |
| FF | ~100K | ~23% |
| BRAM | ~150 | ~42% |
| DSP | ~20 | ~7% |

### 时序

- 系统时钟：100 MHz
- 加速器时钟：100 MHz（同步）
- 预期最大频率：~120 MHz

### 计算性能

对于 64×64 矩阵向量乘法：
- 理论周期数：~64 (每周期一个 MAC)
- 实际周期数：~80 (包含控制开销)
- 延迟：800 ns @ 100MHz
- 等效浮点性能：~5 GFLOPS (考虑量化)

## 扩展方向

### 短期优化

1. **流水线化**
   - 添加流水线寄存器提高频率
   - 目标：150+ MHz

2. **批处理**
   - 支持多向量并行处理
   - 提高吞吐量

3. **DMA 支持**
   - 减少 CPU 干预
   - 自动数据搬移

### 长期规划

1. **多核加速器**
   - 并行处理多个矩阵
   - 扩展到更大网络

2. **可配置精度**
   - 支持 2-bit, 4-bit 量化
   - 灵活的权重编码

3. **完整神经网络支持**
   - 卷积层
   - 激活函数
   - 池化层
